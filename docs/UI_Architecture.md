## 2. 사이드바 상세 설계

### 2.1 구성 요소 (상→하 순서)

```
┌────────────────────────────┐
│  🌀 TyphoonPath            │  ← 로고 (fontSize: 18, fontWeight: 700)
├────────────────────────────┤
│  ① ─── ② ─── ③           │  ← 스텝 인디케이터
├────────────────────────────┤

│  (상태에 따라 hint/좌표 표시)│
├────────────────────────────┤

│  · 중심기압 슬라이더        │

### 2.4 STEP 2 — 기상 조건 슬라이더

```

│                               │
│  중심기압              960 hPa│ ← 값은 조건별 색상

│                               │
│  해수면 온도            29 °C │
│  [20─────────●─────────35]  │

│  ⚡ 강화 가능 구역             │ ← SST 상태 텍스트
│                               │

└──────────────────────────────┘

**슬라이더 값 색상 규칙**

기압: 920 미만 → 빨강(#ef4444) / 960 미만 → 주황(#fb923c) / 이상 → 파랑(#0ea5e9)  
SST: 30 이상 → 빨강 / 27 이상 → 주황 / 미만 → 파랑

## 3. 지도 영역 상세 설계

### 3.1 배너 (상단)

| phase | 배경 강조 색 | 메시지 |
|-------|------------|--------|
| pick | #0ea5e9 (파랑) | 📍 지도를 클릭해 태풍 발생 위치를 설정하세요 |
| config | #f59e0b (황금) | ⚙️ 기상 조건을 설정하고 "경로 예측하기"를 눌러주세요 |
| result | #22c55e (녹색) | ✅ 예측 완료! 파란 선이 예측 경로, 점선이 유사 과거 태풍입니다 |
| loading | - | 🌀 물리 모델로 경로를 계산하고 있습니다... |

### 3.2 지도 레이어 구성 (z-index 순서)

```
4. 시작점 마커 (📍 파란 핀, Permanent Tooltip)
3. 예측 경로 마커 (CircleMarker, 강도 색상)
2. 예측 경로 라인 (Polyline, 파랑 #2563eb, weight: 4)
1. 유사 태풍 라인 (Polyline, 점선, opacity: 0.45)
0. OSM 타일 레이어
```

### 3.3 지도 인터랙션

| phase | 커서 | 클릭 동작 |
|-------|------|---------|
| pick | crosshair | 시작점 설정, phase → config |
| config | grab | 반응 없음 |
| result | grab | 마커 팝업 열기 |

### 3.4 예측 경로 마커 팝업

마커 클릭 시 표시되는 팝업 내용:

```
┌──────────────────┐
│ 예측 6시간 후     │
│ 강도: TY         │
│ 기압: 952 hPa    │
│ 풍속: 55.2 m/s   │
│ 위치: 16.20°N    │
│        133.41°E  │
└──────────────────┘
```

### 3.5 시작점 마커

- 형태: 파란 원형 (36×36px), 흰 테두리, 파란 외곽 링
- 내부: 📍 이모지
- 툴팁: "시작점 15.0°N 135.0°E" (permanent, 상단 표시)

## 4. AI 해설 패널 (하단)

```
┌──────────────────────────────────────────────────────┐
│ 🤖 AI 기상 해설                                       │
│                                                       │
│  이 경로는 960 hPa의 강한 기압과 29°C의 높은 해수면   │
│  온도로 인해 초기 발달 단계에서 북서진을 유지합니다.  │
│  유사 태풍 하이옌(2013)과 유사하게 22~25°N 구간에서  │
│  전향이 예상됩니다. 기압을 940 hPa로 낮추면 이동      │
│  속도가 빨라져 전향 시점이 앞당겨질 수 있습니다.      │
│                                                       │
└──────────────────────────────────────────────────────┘
```

- 최대 높이: 180px (overflow-y: auto)
- 폰트: 13px, 행간 1.7
- 색상: 제목 #0ea5e9 (파랑), 본문 #334155 (다크)
- 표시 조건: phase = 'result' 이고 ai_explanation이 존재할 때

## 5. 상태(State) 관리

### 5.1 전역 상태 목록 (App.tsx)

| 상태명 | 타입 | 초기값 | 설명 |
|--------|------|--------|------|
| phase | 'pick' \| 'config' \| 'result' | 'pick' | 현재 UX 단계 |
| startPoint | {lat, lng} \| null | null | 시작점 좌표 |
| pressure | number | 960 | 중심기압 (hPa) |
| sst | number | 29 | 해수면 온도 (°C) |
| loading | boolean | false | API 요청 중 여부 |
| predictedTrack | PredictedPoint[] | [] | 예측 경로 포인트 목록 |
| analogs | AnalogTyphoon[] | [] | 유사 과거 태풍 목록 |
| explanation | string | '' | AI 기상 설명 텍스트 |
| showAnalogs | boolean | true | 유사 태풍 표시 여부 |

### 5.2 상태 전이 다이어그램

```
초기화
  │
  ▼
phase: 'pick'
  │ 지도 클릭 (handleMapClick)
  ▼
phase: 'config'   ←─── handleReset()
  │ "예측하기" 클릭 (handlePredict)
  │ loading = true
  ▼
API 요청 중 (로딩 배너 표시)
  │ 응답 수신
  ▼
phase: 'result'
  │ "다시 예측하기" (handleReset)
  ▼
phase: 'pick' (전체 초기화)
```

## 7. 반응형 및 접근성 고려사항

- MVP에서는 데스크톱(1280px 이상) 기준으로 설계
- 사이드바는 고정 너비(280px), 지도는 flex: 1로 나머지 공간 점유
- 비활성화 영역은 opacity: 0.35 + pointer-events: none으로 처리
- 로딩 중 예측 버튼 disabled 처리로 중복 요청 방지
- 지도 커서는 단계에 따라 crosshair / grab으로 전환하여 현재 모드 명시

## 8. 에러 처리

| 상황 | 처리 방법 |
|------|----------|
| API 요청 실패 (네트워크 오류) | alert('예측 실패: ' + error.message) |
| 유사 태풍 0개 | analogs 빈 배열, 지도에 표시 없음 |
| AI 설명 생성 실패 | 빈 문자열 반환, 패널 미표시 |
| 시작점 미설정 후 예측 버튼 | 버튼 disabled 처리로 원천 차단 |
