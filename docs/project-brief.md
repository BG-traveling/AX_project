## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | TyphoonPath — 태풍경로예측 |
| 유형 | 체험형 교육 서비스 (웹) |
| 개발 기간 | 3일 (DAY 1 ~ DAY 3) |
| 담당자 | 김동현 |

### 한 줄 소개
사용자가 태풍의 기압, 온도 등 주요 기상 변수를 직접 조작하여 경로를 예측하고, AI로부터 실제 경로와의 비교 피드백을 받는 **체험형 태풍 경로 학습 서비스**입니다.

---

## 2. 문제 정의

태풍 경로 예측의 원리는 기상학적으로 복잡하여 일반인이 직관적으로 이해하기 어렵습니다. 기존 기상 교육 콘텐츠는 수동적 정보 전달에 그치며, 사용자가 직접 변수를 조작해보고 결과를 체험하는 인터랙티브 학습 도구가 부재합니다.

---

## 3. 목표

- 사용자가 태풍 경로의 원리를 **직관적으로 이해**할 수 있도록 한다
- 기상 변수(기압, 온도 등)가 태풍 이동에 미치는 영향을 **체험을 통해 학습**하도록 돕는다
- AI 피드백을 통해 예측 결과에 대한 **즉각적이고 교육적인 피드백**을 제공한다

---

## 4. 핵심 기능 요약

### 4.1 과거 태풍 시각화
- 1951년 ~ 2024년 과거 태풍 경로 데이터를 지도 위에 시각화
- 연도 및 태풍 이름 필터링

### 4.2 변수 조작
- 기압, 온도 등 기상 변수를 슬라이더/인풋으로 직접 조정
- 조작 값에 따른 예상 경로 실시간 반영
- 사용자가 예상 경로를 지도 위에 직접 그려 제출

### 4.3 AI 피드백
- 사용자 예측 경로 vs 실제 경로 비교 분석
- Claude API를 통한 예측 오차 원인 및 학습 포인트 설명

---

## 5. 기술 스택

| 구분 | 기술 |
|------|------|
| Frontend | React 18, TypeScript, Vite, Leaflet.js |
| Backend | FastAPI (Python) |
| AI | Claude API (Anthropic) |
| 캐싱 | Redis |
| 데이터 출처 | 기상청 기상자료개방포털, 공공데이터포털 |
| 데이터 범위 | 1951년 ~ 2024년 태풍 통합 데이터 |
| 배포 | Vercel (Frontend), Railway (Backend) |

---

## 6. 사용자 흐름

```
태풍 선택 (연도/이름 필터)
    ↓
변수 조작 (기압, 온도 슬라이더)
    ↓
경로 예측 제출 (지도 위 직접 드로잉)
    ↓
실제 경로 비교 & AI 피드백 수신
```

---

## 7. 일별 개발 계획

| 일차 | 목표 |
|------|------|
| DAY 1 | 기초 구축 및 데이터 연동 (환경 설정, API 연동, Redis 캐싱) |
| DAY 2 | 핵심 기능 개발 (태풍 선택, Leaflet 시각화, 변수 조작 UI, 경로 제출) |
| DAY 3 | AI 피드백 연동 및 최종 점검 (Claude API, UX 개선, 배포) |

---

## 8. 성공 기준

- 1951~2024년 태풍 데이터 정상 조회 및 지도 시각화
- 기상 변수 조작 → 예상 경로 실시간 반영
- 사용자 예측 제출 → Claude API 피드백 정상 수신
- Vercel + Railway 배포 완료
