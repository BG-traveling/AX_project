# Story: DAY 2 — 핵심 기능 개발 및 시각화

**BMAD 단계:** Phase 4 - Development (Scrum Master)  
**목표:** 사용자가 태풍 선택 → 변수 조작 → 경로 제출까지 완성

---

## Story 2-1: 태풍 선택 UI

**As a** 사용자  
**I want to** 연도와 태풍 이름으로 원하는 태풍을 선택할 수 있다  
**So that** 관심 있는 태풍의 데이터를 볼 수 있다

### 작업 체크리스트
- [ ] `TyphoonSelector.tsx` — 연도 드롭다운 구현
- [ ] `TyphoonSelector.tsx` — 태풍 이름 드롭다운 구현 (연도 선택 후 목록 갱신)
- [ ] `useTyphoonData.ts` — 선택된 태풍 ID로 경로 데이터 fetch 훅 구현
- [ ] 선택 시 지도에 해당 태풍 경로 표시

**완료 기준:** 연도/이름 선택 → 지도에 경로 표시

---

## Story 2-2: Leaflet 지도 태풍 경로 시각화

**As a** 사용자  
**I want to** 선택한 태풍의 실제 이동 경로를 지도에서 확인할 수 있다  
**So that** 경로 패턴을 직관적으로 파악할 수 있다

### 작업 체크리스트
- [ ] `TyphoonMap.tsx` — react-leaflet 기본 지도 설정 (태평양 중심, 줌레벨 4)
- [ ] `TyphoonTrack.tsx` — 태풍 경로 Polyline 렌더링
- [ ] 경로 포인트 마커 표시 (기압/풍속 팝업 포함)
- [ ] 태풍 강도에 따른 마커 색상 구분 (TD/TS/TY/STY)
- [ ] `GET /api/typhoons/{id}/track` 백엔드 라우터 구현

**완료 기준:** 태풍 경로가 지도 위에 색상 구분된 마커와 함께 표시

---

## Story 2-3: 변수 조작 패널 (기압/온도 슬라이더)

**As a** 사용자  
**I want to** 기압과 온도 슬라이더를 조정하여 경로 변화를 체험한다  
**So that** 기상 변수와 태풍 경로의 관계를 직접 느낄 수 있다

### 작업 체크리스트
- [ ] `VariablePanel.tsx` — 기압 슬라이더 (850~1010 hPa)
- [ ] `VariablePanel.tsx` — 온도 슬라이더 (해수면 온도 범위)
- [ ] 슬라이더 값 변화 → 간단한 경로 변형 시뮬레이션 반영
  - 기압 낮을수록 더 강한 경로 편향 적용 (단순 선형 모델)
  - 온도 높을수록 북상 속도 증가
- [ ] 현재 변수값을 UI에 숫자로 표시
- [ ] 변수 초기화 버튼

**완료 기준:** 슬라이더 조작 시 지도의 예상 경로가 실시간 변화

---

## Story 2-4: 사용자 경로 드로잉 및 제출

**As a** 사용자  
**I want to** 지도 위에 직접 예상 태풍 경로를 그려서 제출할 수 있다  
**So that** 나의 예측을 실제 경로와 비교할 수 있다

### 작업 체크리스트
- [ ] `DrawingLayer.tsx` — 지도 클릭으로 경로 포인트 추가
- [ ] 드로잉 경로를 폴리라인으로 표시 (점선, 다른 색상)
- [ ] 경로 초기화 버튼 구현
- [ ] 제출 버튼 → 위경도 배열을 백엔드로 POST
- [ ] 제출 완료 시 피드백 패널로 전환

**완료 기준:** 지도에 경로 그리기 → 제출 시 API 호출 성공
