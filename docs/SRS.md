# 소프트웨어 요구사항 명세서 (SRS)

**프로젝트명**: TyphoonPath — 태풍 경로 예측 시뮬레이터  
**버전**: 2.0  
**작성일**: 2026-06-29  
**작성자**: 김동현  
**상태**: 진행 중

---

## 1. 개요

### 1.1 목적

이 문서는 TyphoonPath 웹 서비스의 기능 요구사항, 비기능 요구사항, 시스템 아키텍처를 정의한다. 개발자와 이해관계자가 공통된 이해를 바탕으로 프로젝트를 진행할 수 있도록 작성되었다.

### 1.2 서비스 개요

TyphoonPath는 사용자가 태풍 발생 위치와 기상 초기 조건을 설정하면, 서비스가 물리 기반 수치 모델로 예측 경로를 자동 생성하고, 유사 과거 태풍 사례와 비교하여 AI가 경로 이유를 설명해 주는 교육용 기상 시뮬레이터다.

### 1.3 배경 및 필요성

초기 설계는 사용자가 직접 경로를 그리고 실제 경로와 비교하는 방식이었으나, 서비스 핵심 가치 문제가 발견되었다. 화면에 실제 경로가 이미 표시된 상태에서 사용자가 예측을 그리는 방식은 교육적 긴장감을 만들지 못하고, 서비스를 이용할 동기도 사라진다는 판단 하에 현재 구조로 전환하였다.

새 구조의 핵심은 다음과 같다. 사용자가 초기 조건을 설정하면 **서비스가 예측**하고, 사용자는 기상 조건을 조작하면서 경로가 어떻게 달라지는지 직접 확인한다. 이를 통해 기압, 해수면 온도, 위도가 태풍 경로에 미치는 영향을 직관적으로 학습할 수 있다.

### 1.4 용어 정의

| 용어 | 설명 |
|------|------|
| IBTrACS | International Best Track Archive for Climate Stewardship. WMO 공식 태풍 기록 데이터셋 |
| SST | Sea Surface Temperature, 해수면 온도 |
| Beta drift | 저위도에서 지구 자전(코리올리 효과)에 의한 태풍의 북서 방향 편향 이동 |
| 전향 (Recurvature) | 태풍이 북위 25~28도 부근에서 편서풍대에 진입하여 북동쪽으로 방향을 트는 현상 |
| Analog Method | 초기 조건이 유사한 과거 사례를 탐색하여 미래 경로를 추정하는 기법 |
| STY | Super Typhoon, 최대 풍속 33 m/s 이상 |
| TY | Typhoon, 최대 풍속 25~32 m/s |
| TS | Tropical Storm, 최대 풍속 17~24 m/s |
| TD | Tropical Depression, 최대 풍속 17 m/s 미만 |

## 3. 기능 요구사항

### 3.1 핵심 기능 목록

| ID | 기능명 | 우선순위 |
|----|--------|---------|
| F-01 | 시작점 설정 (지도 클릭) | 필수 |
| F-02 | 기상 초기 조건 설정 (기압, SST) | 필수 |
| F-03 | 물리 모델 경로 예측 | 필수 |
| F-04 | 유사 과거 태풍 탐색 및 표시 | 필수 |
| F-05 | AI 경로 설명 생성 | 필수 |
| F-06 | 강도 강세별 색상 마커 | 권장 |
| F-07 | 예측 결과 초기화 및 재예측 | 필수 |

### 3.2 F-01: 시작점 설정

사용자가 지도를 클릭하면 해당 위경도 좌표가 태풍 발생 시작점으로 설정된다.

- 서태평양 영역(위도 0~40°N, 경도 100~180°E)을 기본 뷰로 제공한다.
- 시작점 설정 후 파란 핀 마커가 표시되며, 좌표가 사이드바에 표시된다.
- 시작점은 예측 실행 전까지 "변경" 버튼으로 재설정 가능하다.

### 3.3 F-02: 기상 초기 조건 설정

사용자는 슬라이더를 통해 두 가지 기상 변수를 설정한다.

**중심기압 (hPa)**
- 범위: 850 ~ 1010 hPa
- 기본값: 960 hPa
- 1010 hPa에 가까울수록 약한 저기압 / 850 hPa에 가까울수록 초강력 태풍

**해수면 온도 (SST, °C)**
- 범위: 20 ~ 35°C
- 기본값: 29°C
- 26°C 이상 구역에서 태풍이 발달하고, 28°C 이상에서 급격히 강화된다.

조건 변경 시 사이드바에 예상 강도 레이블(TD / TS / TY / STY)이 실시간으로 업데이트된다.

### 3.4 F-03: 물리 모델 경로 예측

"경로 예측하기" 버튼을 누르면 백엔드 물리 모델이 경로를 생성한다.

**알고리즘 개요**

예측은 6시간 간격으로 최대 20스텝(120시간, 5일) 시뮬레이션을 수행한다.

*조향 벡터*: 위도에 따라 이동 방향을 결정한다.
- 위도 15°N 미만: 북서진 (Beta drift 지배)
- 위도 15~22°N: 북북서진 (무역풍 + Beta drift)
- 위도 22~28°N: 전향 구간 (편서풍대 진입)
- 위도 28°N 초과: 북동진 (편서풍 지배)

*강도 변화*: SST와 위도에 따라 기압이 변화한다.
- SST ≥ 28°C, 위도 < 25°N: 발달 (기압 하강)
- SST < 26°C 또는 위도 > 35°N: 약화 (기압 상승)
- 기압 1005 hPa 도달 시 소멸로 처리

*압력-풍속 변환*:
```
wind_ms = min(85.0, (1010 - pressure)^0.644 × 3.92)
```

예측 결과는 지도에 파란 실선으로 표시되며, 각 관측점은 강도에 따라 색상이 다른 원 마커로 표시된다. 마커 클릭 시 해당 시점의 상세 정보(시간, 위치, 기압, 풍속, 강도)가 팝업으로 나타난다.

### 3.5 F-04: 유사 과거 태풍 탐색 (Analog Method)

예측과 동시에 IBTrACS 데이터에서 조건이 유사한 과거 태풍 최대 3개를 탐색한다.

**유사도 점수 계산**

```
score = pos_score × 0.5 + pres_score × 0.3 + season_score × 0.2
```

| 항목 | 가중치 | 기준 |
|------|--------|------|
| 시작 위치 | 50% | 거리 < 1,500km |
| 중심기압 | 30% | 차이 < 80 hPa |
| 계절 | 20% | 월 차이 ≤ 3 |

유사 태풍 경로는 반투명 점선으로 표시된다 (3개 각각 다른 색상). 마우스 오버 시 태풍명, 발생 연도, 유사도(%)가 툴팁으로 표시된다.

### 3.6 F-05: AI 경로 설명 생성

예측 완료 후 Claude Haiku가 경로에 대한 기상학적 설명을 자동 생성한다.

설명에는 다음 세 가지 요소가 포함된다.
1. 이 경로가 예측된 주요 기상 원인 (기압, SST, 위도 조향력)
2. 유사 과거 태풍과의 공통점 한 가지
3. 변수를 바꾸면 경로가 어떻게 달라질지 학습 힌트

설명은 화면 하단 패널에 표시되며, 200자 이내의 한국어로 작성된다.

### 3.7 F-07: 재예측

"다시 예측하기" 버튼으로 전체 상태를 초기화하고, STEP 1(시작점 선택)부터 다시 시작할 수 있다.

## 4. UX 플로우 — 3단계 구조

```
STEP 1: 시작점 선택
  ↓ 지도 클릭
STEP 2: 기상 조건 설정
  ↓ 슬라이더 조작 → "경로 예측하기" 클릭
STEP 3: 결과 확인
  - 예측 경로 (파란 실선 + 강도 마커)
  - 유사 과거 태풍 (점선, 체크박스로 토글)
  - AI 기상 설명 (하단 패널)
  ↓ "다시 예측하기" 클릭
STEP 1 (반복)
```

각 단계는 사이드바 상단의 스텝 인디케이터(1→2→3)로 시각화된다. 이전 단계가 완료되지 않으면 다음 단계 컨트롤은 비활성화(opacity 0.35)된다.

## 5. 비기능 요구사항

### 5.1 성능

| 항목 | 목표값 |
|------|--------|
| 물리 모델 연산 | < 100ms (서버 내) |
| 유사 태풍 탐색 | < 500ms (2,645개 전체 스캔) |
| AI 설명 생성 | < 5초 (Anthropic API 응답 포함) |
| 전체 예측 응답 | < 8초 |

### 5.2 데이터

- typhoons.json은 서버 기동 시 `lru_cache`로 메모리에 한 번만 로드한다.
- 데이터 크기: 약 16.5MB, 172,206개 트랙 포인트.
- 별도 데이터베이스 없이 파일 기반으로 서빙한다.

### 5.3 접근성 및 환경

- 모바일 최적화는 MVP 범위에 포함하지 않는다.
- 최신 크롬/파이어폭스/엣지 브라우저를 지원한다.
- 지도 타일은 OpenStreetMap을 사용한다 (무료, 오픈소스).

### 5.4 보안

- Anthropic API 키는 `.env` 파일로 관리하며 프론트엔드에 노출하지 않는다.
- CORS는 개발 단계에서 localhost:5173, localhost:3000만 허용한다.

## 6. API 명세

### POST /api/predict

태풍 경로를 예측하고 유사 과거 태풍과 AI 설명을 함께 반환한다.

**Request Body**

```json
{
  "start_lat": 15.0,
  "start_lng": 135.0,
  "pressure": 960,
  "sst": 29.0,
  "month": 8
}
```

| 필드 | 타입 | 범위 | 설명 |
|------|------|------|------|
| start_lat | float | 0 ~ 40 | 시작 위도 |
| start_lng | float | 100 ~ 180 | 시작 경도 |
| pressure | float | 850 ~ 1010 | 중심기압 (hPa) |
| sst | float | 20 ~ 35 | 해수면 온도 (°C) |
| month | int | 1 ~ 12 | 예측 월 (계절성 계산용) |

**Response Body**

```json
{
  "predicted_track": [
    {
      "lat": 15.83,
      "lng": 133.41,
      "pressure": 955.2,
      "wind_ms": 48.3,
      "intensity": "TY",
      "hour": 6
    }
  ],
  "analogs": [
    {
      "id": "2013296N07148",
      "name_en": "HAIYAN",
      "year": 2013,
      "similarity": 0.87,
      "track": [{"lat": 14.5, "lng": 135.2}]
    }
  ],
  "ai_explanation": "이 경로는 960 hPa의 강한 기압과 29°C의 높은 해수면 온도..."
}
```

## 7. 화면 구성

### 7.1 레이아웃

전체 화면은 좌측 사이드바(280px)와 우측 지도 영역으로 구성된다.

**사이드바**
- 로고 ("🌀 TyphoonPath")
- 스텝 인디케이터 (1→2→3)
- STEP 1: 시작점 좌표 표시
- STEP 2: 기압 슬라이더, SST 슬라이더, 예측 버튼
- STEP 3: 예측 요약 배지 (강도, 기압, 풍속, 예측 기간), 유사 태풍 목록, 재예측 버튼
- 하단: 강도 범례

**지도 영역**
- 상단 배너: 현재 단계 안내 메시지
- 지도: Leaflet, OSM 타일
- 하단 패널: AI 기상 해설 (결과 단계에서만 표시)
